# Plan: MVP tgcasino

**Goal:** Telegram-бот казино с виртуальной валютой "Сабинка" — кубик 🎲 и слот 🎰, фиксированные ставки, промокод.
**Approach:** Монолит на Go, чистые пакеты внутри internal/
**Stack:** go-telegram-bot-api v5, PostgreSQL, golang-migrate, godotenv
**Plan file:** docs/plans/2026-03-28-mvp.md

---

## Обзор задач

1. Инициализация модуля и структуры папок
2. Конфиг (чтение env)
3. БД: схема и миграции
4. Storage: репозитории (users, transactions)
5. Wallet: баланс и операции
6. Payment: промокоды (скелет)
7. Game: логика кубика и слота
8. Bot: handlers и роутинг апдейтов

```go
import (
    "log"
    "os"

    "github.com/joho/godotenv"
)

type Config struct {
    BotToken string
    DBUrl    string
}

func Load() *Config {
    if err := godotenv.Load(); err != nil {
        log.Println("файл .env не найден, читаем из окружения")
    }

    return &Config{
        BotToken: mustGet("BOT_TOKEN"),
        DBUrl:    mustGet("DATABASE_URL"),
    }
}

func mustGet(key string) string {
    v := os.Getenv(key)
    if v == "" {
        log.Fatalf("переменная окружения %s не задана", key)
    }
    return v
}
```

## Task 3: Миграции и схема БД

**Что делаем:** SQL-файлы для создания таблиц, функция запуска миграций.

**Файлы:**
- `backend/migrations/001_init.up.sql`
- `backend/migrations/001_init.down.sql`
- `backend/internal/storage/migrate.go`

**`migrations/001_init.up.sql`:**
```sql
CREATE TABLE users (
    id         BIGINT PRIMARY KEY,  -- telegram user id
    username   TEXT,
    balance    BIGINT NOT NULL DEFAULT 0,  -- в сабинках
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE transactions (
    id         BIGSERIAL PRIMARY KEY,
    user_id    BIGINT NOT NULL REFERENCES users(id),
    amount     BIGINT NOT NULL,       -- положительное = начисление, отрицательное = списание
    reason     TEXT NOT NULL,          -- "promo", "bet", "win", "daily"
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE promo_codes (
    code       TEXT PRIMARY KEY,
    amount     BIGINT NOT NULL,
    used_by    BIGINT REFERENCES users(id),
    used_at    TIMESTAMPTZ
);
```

**`migrations/001_init.down.sql`:**
```sql
DROP TABLE IF EXISTS transactions;
DROP TABLE IF EXISTS promo_codes;
DROP TABLE IF EXISTS users;
```

**`internal/storage/migrate.go`:**
```go
package storage

import (
    "log"

    "github.com/golang-migrate/migrate/v4"
    _ "github.com/golang-migrate/migrate/v4/database/postgres"
    _ "github.com/golang-migrate/migrate/v4/source/file"
)

func RunMigrations(dbURL string) {
    m, err := migrate.New("file://migrations", dbURL)
    if err != nil {
        log.Fatalf("migrate.New: %v", err)
    }
    if err := m.Up(); err != nil && err != migrate.ErrNoChange {
        log.Fatalf("migrate.Up: %v", err)
    }
    log.Println("миграции применены")
}
```

**Почему:** миграции — это версионированная история схемы БД. `up` — применить, `down` — откатить. `golang-migrate` читает файлы из папки и применяет только те, что ещё не применены. `ErrNoChange` — это норма, не ошибка.

**Коммит:** `db: initial schema and migrations`

**`internal/storage/user.go`:**
```go
package storage

import (
    "database/sql"
)

type User struct {
    ID       int64
    Username string
    Balance  int64
}

// GetOrCreate — находит пользователя или создаёт нового с нулевым балансом
func GetOrCreate(db *sql.DB, id int64, username string) (*User, error) {
    u := &User{}
    err := db.QueryRow(
        `INSERT INTO users (id, username) VALUES ($1, $2)
         ON CONFLICT (id) DO UPDATE SET username = EXCLUDED.username
         RETURNING id, username, balance`,
        id, username,
    ).Scan(&u.ID, &u.Username, &u.Balance)
    return u, err
}

func GetBalance(db *sql.DB, userID int64) (int64, error) {
    var balance int64
    err := db.QueryRow(`SELECT balance FROM users WHERE id = $1`, userID).Scan(&balance)
    return balance, err
}

func UpdateBalance(db *sql.DB, userID int64, delta int64) error {
    _, err := db.Exec(
        `UPDATE users SET balance = balance + $1 WHERE id = $2`,
        delta, userID,
    )
    return err
}
```

**`internal/storage/transaction.go`:**
```go
package storage

import "database/sql"

func AddTransaction(db *sql.DB, userID int64, amount int64, reason string) error {
    _, err := db.Exec(
        `INSERT INTO transactions (user_id, amount, reason) VALUES ($1, $2, $3)`,
        userID, amount, reason,
    )
    return err
}
```

**Почему:** `ON CONFLICT DO UPDATE` — это upsert. Первый раз создаёт, последующие — обновляют username (он может меняться). Транзакции пишем отдельной записью каждый раз — это аудит лог всех движений денег.

**Коммит:** `storage: user and transaction repositories`

## Task 5: Wallet — операции с балансом

**Что делаем:** бизнес-логика кошелька поверх storage. Здесь проверки (достаточно ли денег), атомарные операции.

**Файлы:**
- `backend/internal/wallet/wallet.go`

```go
package wallet

import (
    "database/sql"
    "errors"

    "tgcasino/internal/storage"
)

var ErrInsufficientFunds = errors.New("недостаточно сабинок")

// Debit — списать amount со счёта. Атомарно проверяет и списывает.
func Debit(db *sql.DB, userID int64, amount int64, reason string) error {
    tx, err := db.Begin()
    if err != nil {
        return err
    }
    defer tx.Rollback()

    var balance int64
    err = tx.QueryRow(
        `SELECT balance FROM users WHERE id = $1 FOR UPDATE`, userID,
    ).Scan(&balance)
    if err != nil {
        return err
    }

    if balance < amount {
        return ErrInsufficientFunds
    }

    if _, err = tx.Exec(
        `UPDATE users SET balance = balance - $1 WHERE id = $2`, amount, userID,
    ); err != nil {
        return err
    }

    if _, err = tx.Exec(
        `INSERT INTO transactions (user_id, amount, reason) VALUES ($1, $2, $3)`,
        userID, -amount, reason,
    ); err != nil {
        return err
    }

    return tx.Commit()
}

// Credit — начислить amount на счёт.
func Credit(db *sql.DB, userID int64, amount int64, reason string) error {
    if err := storage.UpdateBalance(db, userID, amount); err != nil {
        return err
    }
    return storage.AddTransaction(db, userID, amount, reason)
}
```

**Почему:** `FOR UPDATE` блокирует строку на время транзакции — если два запроса придут одновременно, второй подождёт. Это защита от гонки когда у игрока 100 сабинок и он делает два запроса по 100 одновременно.

**Коммит:** `wallet: debit and credit with atomic balance check`

**`internal/payment/promo.go`:**
```go
package payment

import (
    "database/sql"
    "errors"

    "tgcasino/internal/wallet"
)

var (
    ErrPromoNotFound = errors.New("промокод не найден")
    ErrPromoUsed     = errors.New("промокод уже использован")
)

func ApplyPromo(db *sql.DB, userID int64, code string) (int64, error) {
    var amount int64
    var usedBy sql.NullInt64

    err := db.QueryRow(
        `SELECT amount, used_by FROM promo_codes WHERE code = $1`, code,
    ).Scan(&amount, &usedBy)
    if err == sql.ErrNoRows {
        return 0, ErrPromoNotFound
    }
    if err != nil {
        return 0, err
    }
    if usedBy.Valid {
        return 0, ErrPromoUsed
    }

    _, err = db.Exec(
        `UPDATE promo_codes SET used_by = $1, used_at = NOW() WHERE code = $2`,
        userID, code,
    )
    if err != nil {
        return 0, err
    }

    return amount, wallet.Credit(db, userID, amount, "promo:"+code)
}
```

## Task 7: Game — кубик и слот

**Что делаем:** логика расчёта выигрыша по результату Telegram dice.

**Файлы:**
- `backend/internal/game/dice.go`
- `backend/internal/game/slots.go`

**`internal/game/dice.go`:**
```go
package game

// Telegram возвращает значение 1-6 для 🎲
// Множители выигрыша:
//   1 → 0x (проигрыш)
//   2 → 0x
//   3 → 1x (возврат ставки)
//   4 → 2x
//   5 → 3x
//   6 → 5x

var diceMultipliers = map[int]int64{
    1: 0,
    2: 0,
    3: 1,
    4: 2,
    5: 3,
    6: 5,
}

// DiceResult возвращает выигрыш в сабинках (0 = проигрыш)
func DiceResult(bet int64, diceValue int) int64 {
    mult, ok := diceMultipliers[diceValue]
    if !ok {
        return 0
    }
    return bet * mult
}
```

**`internal/game/slots.go`:**
```go
package game

// Telegram 🎰 возвращает значение 1-64
// Джекпот = 64 (777)
// Выигрыш средний = 22, 43 (два одинаковых символа, по документации tg)
// Остальное = проигрыш

func SlotsResult(bet int64, slotValue int) int64 {
    switch slotValue {
    case 64: // джекпот 777
        return bet * 10
    case 22, 43: // два совпадения
        return bet * 2
    default:
        return 0
    }
}
```

**Почему:** логика игр полностью отделена от Telegram и БД. Это просто функции int→int64 — их легко тестировать и менять мультипликаторы не трогая остальной код. Значения для слота (22, 43, 64) — официальные из документации Telegram Bot API.

**Коммит:** `game: dice and slots result logic`

```go
package bot

import (
    "database/sql"
    "log"

    tgbotapi "github.com/go-telegram-bot-api/telegram-bot-api/v5"
)

type Bot struct {
    api *tgbotapi.BotAPI
    db  *sql.DB
}

func New(token string, db *sql.DB) *Bot {
    api, err := tgbotapi.NewBotAPI(token)
    if err != nil {
        log.Fatalf("tgbotapi.NewBotAPI: %v", err)
    }
    log.Printf("бот запущен: @%s", api.Self.UserName)
    return &Bot{api: api, db: db}
}

func (b *Bot) Start() {
    u := tgbotapi.NewUpdate(0)
    u.Timeout = 60
    updates := b.api.GetUpdatesChan(u)

    for update := range updates {
        if update.Message == nil {
            continue
        }
        b.route(update.Message)
    }
}
```

**`internal/bot/handlers.go`:**
```go
package bot

import (
    "fmt"

    tgbotapi "github.com/go-telegram-bot-api/telegram-bot-api/v5"
    "tgcasino/internal/payment"
    "tgcasino/internal/storage"
    "tgcasino/internal/wallet"
)

func (b *Bot) route(msg *tgbotapi.Message) {
    switch msg.Command() {
    case "start":
        b.handleStart(msg)
    case "balance":
        b.handleBalance(msg)
    case "promo":
        b.handlePromo(msg)
    case "dice":
        b.handleGame(msg, "dice")
    case "slots":
        b.handleGame(msg, "slots")
    }
}

func (b *Bot) send(chatID int64, text string) {
    msg := tgbotapi.NewMessage(chatID, text)
    b.api.Send(msg)
}

func (b *Bot) handleStart(msg *tgbotapi.Message) {
    user, err := storage.GetOrCreate(b.db, msg.From.ID, msg.From.UserName)
    if err != nil {
        b.send(msg.Chat.ID, "ошибка при регистрации")
        return
    }
    b.send(msg.Chat.ID, fmt.Sprintf(
        "Привет, %s! Твой баланс: %d Сабинок 💰\n\nКоманды:\n/balance — баланс\n/promo <код> — активировать промокод\n/dice — кубик 🎲\n/slots — слот 🎰",
        user.Username, user.Balance,
    ))
}

func (b *Bot) handleBalance(msg *tgbotapi.Message) {
    balance, err := storage.GetBalance(b.db, msg.From.ID)
    if err != nil {
        b.send(msg.Chat.ID, "пользователь не найден, напиши /start")
        return
    }
    b.send(msg.Chat.ID, fmt.Sprintf("Баланс: %d Сабинок 💰", balance))
}

func (b *Bot) handlePromo(msg *tgbotapi.Message) {
    code := msg.CommandArguments()
    if code == "" {
        b.send(msg.Chat.ID, "использование: /promo <код>")
        return
    }

    amount, err := payment.ApplyPromo(b.db, msg.From.ID, code)
    switch err {
    case nil:
        b.send(msg.Chat.ID, fmt.Sprintf("Промокод активирован! +%d Сабинок 🎉", amount))
    case payment.ErrPromoNotFound:
        b.send(msg.Chat.ID, "промокод не найден")
    case payment.ErrPromoUsed:
        b.send(msg.Chat.ID, "промокод уже использован")
    default:
        b.send(msg.Chat.ID, "ошибка при активации")
    }
}

func (b *Bot) handleGame(msg *tgbotapi.Message, gameType string) {
    _, err := storage.GetOrCreate(b.db, msg.From.ID, msg.From.UserName)
    if err != nil {
        b.send(msg.Chat.ID, "сначала напиши /start")
        return
    }

    // показываем кнопки выбора ставки
    keyboard := tgbotapi.NewInlineKeyboardMarkup(
        tgbotapi.NewInlineKeyboardRow(
            tgbotapi.NewInlineKeyboardButtonData("10", gameType+":10"),
            tgbotapi.NewInlineKeyboardButtonData("50", gameType+":50"),
            tgbotapi.NewInlineKeyboardButtonData("100", gameType+":100"),
            tgbotapi.NewInlineKeyboardButtonData("500", gameType+":500"),
        ),
    )
    reply := tgbotapi.NewMessage(msg.Chat.ID, "Выбери ставку (Сабинки):")
    reply.ReplyMarkup = keyboard
    b.api.Send(reply)
}
```

**`internal/bot/game_handler.go`:**
```go
package bot

import (
    "fmt"
    "strconv"
    "strings"

    tgbotapi "github.com/go-telegram-bot-api/telegram-bot-api/v5"
    "tgcasino/internal/game"
    "tgcasino/internal/wallet"
)

// handleCallback — обрабатывает нажатие кнопки ставки
func (b *Bot) handleCallback(cb *tgbotapi.CallbackQuery) {
    parts := strings.SplitN(cb.Data, ":", 2)
    if len(parts) != 2 {
        return
    }
    gameType, betStr := parts[0], parts[1]

    bet, err := strconv.ParseInt(betStr, 10, 64)
    if err != nil {
        return
    }

    // снимаем ставку
    if err := wallet.Debit(b.db, cb.From.ID, bet, "bet:"+gameType); err != nil {
        if err == wallet.ErrInsufficientFunds {
            b.api.AnswerCallbackQuery(tgbotapi.NewCallback(cb.ID, "недостаточно сабинок"))
        }
        return
    }
    b.api.AnswerCallbackQuery(tgbotapi.NewCallback(cb.ID, ""))

    // отправляем emoji
    var diceEmoji string
    switch gameType {
    case "dice":
        diceEmoji = "🎲"
    case "slots":
        diceEmoji = "🎰"
    }

    diceMsg := tgbotapi.NewDice(cb.Message.Chat.ID)
    diceMsg.Emoji = diceEmoji
    sent, err := b.api.Send(diceMsg)
    if err != nil || sent.Dice == nil {
        // возвращаем ставку если что-то пошло не так
        wallet.Credit(b.db, cb.From.ID, bet, "refund:"+gameType)
        return
    }

    // считаем выигрыш
    var winAmount int64
    switch gameType {
    case "dice":
        winAmount = game.DiceResult(bet, sent.Dice.Value)
    case "slots":
        winAmount = game.SlotsResult(bet, sent.Dice.Value)
    }

    if winAmount > 0 {
        wallet.Credit(b.db, cb.From.ID, winAmount, "win:"+gameType)
        b.send(cb.Message.Chat.ID, fmt.Sprintf("Выигрыш: %d Сабинок 🎉", winAmount))
    } else {
        b.send(cb.Message.Chat.ID, fmt.Sprintf("Не повезло. Ставка %d Сабинок сгорела.", bet))
    }
}
```

**Обнови `bot.go` — добавь обработку CallbackQuery в `Start()`:**
```go
for update := range updates {
    if update.Message != nil {
        b.route(update.Message)
    } else if update.CallbackQuery != nil {
        b.handleCallback(update.CallbackQuery)
    }
}
```

**Почему:** CallbackQuery — это нажатие на inline-кнопку. Данные кнопки (`cb.Data`) содержат тип игры и ставку через `:`. Ставку снимаем ДО броска — это честно и не даёт возможности отменить проигрыш.
